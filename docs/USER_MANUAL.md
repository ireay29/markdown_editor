# ユーザーマニュアル

この度は「マークダウンエディター」をご利用いただきありがとうございます。本エディターは、通常のMarkdown編集機能に加え、LLMプロンプト作成を支援する強力な独自機能を備えています。

## 1. 基本操作

### ファイル操作

- **新規作成**: サイドバー上部の「新規作成」ボタンから新しいファイルを作成します。
- **読み込み**: 「ファイルを開く」からPC内の `.md` ファイルを選択します。
- **保存**: 編集内容は自動保存機能により安全に保管されますが、手動での名前を付けて保存も可能です。

### 編集とプレビュー

- **3ペイン構成**: 左から「サイドバー（アウトライン/ブロック）」、「エディター（編集）」、「プレビュー（表示）」の構成になっています。
- **リアルタイム反映**: エディターでの書き込みは即座にプレビューへ反映されます。

---

## 2. 独自機能ガイド

### ブロック管理システム（独自機能1）

Markdownを単なるテキストではなく、意味のある「ブロック（見出し、段落、コード等）」単位で管理できます。

- **並び替え**: サイドバーの「Blocks」タブから、ブロックをドラッグ＆ドロップすることで、テキストの順序を簡単に入れ替えることができます。
- **セクション管理**: 長文のドキュメント構成を練り直す際に非常に便利です。

### 変数システム（独自機能4）

LLMへのプロンプト作成を効率化するための機能です。

- **変数の定義**: Markdown内に `{{user_name}}` のように二重中括弧で記述すると、自動的に「変数」として抽出されます。
- **一括置換**: サイドバーの「Variables」パネルで変数の値を入力すると、ドキュメント全体の対応箇所の表示が一括で更新されます。
- **テンプレート活用**: 定型文をテンプレートとして保存し、変数値だけを入れ替えてプロンプトを量産できます。

### 構文チェック・リンター（独自機能3）

- **リアルタイム検出**: Markdownの書き方に間違いがある場合、エディター上に赤い下線が表示されます。
- **エラー詳細**: 下線部にカーソルを合わせるか、画面下部のエラーパネルを確認することで、ミスの内容を把握できます。

---

## 3. カスタイマイズと設定

### 表示設定

- **ダークモード**: システム設定に合わせた自動切り替え、または手動での「ライト/ダーク」選択が可能です。
- **フォント**: 設定パネルからフォントサイズや行間を、自分の読みやすい設定に調整できます。

### キーボードショートカット

- `Ctrl/Cmd + S`: 保存
- `Ctrl/Cmd + F`: 検索
- `Ctrl/Cmd + Z`: 元に戻す
- `Ctrl/Cmd + /`: サイドバー/プレビューの展開・折りたたみ

---

## 4. トラブルシューティング

- **画面が白くなった**: アプリを再起動するか、設定パネルの「キャッシュクリア」を試してください。
- **プレビューが更新されない**: 数秒待っても反映されない場合は、Markdownの構文エラー（独自機能3）が原因の可能性があります。パネルを確認してください。
