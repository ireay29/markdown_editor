# 機能仕様書 (Feature Specification Document)

## 1. 目的 (Purpose)
このドキュメントは、高機能Markdownエディターの主要な機能とその詳細を明確に記述することを目的としています。これにより、開発者、テスター、および将来のユーザーが本エディターの機能と使い方を正確に理解できるようになります。

## 2. 概要 (Overview)
本Markdownエディターは、Tauri、React、TypeScriptを基盤として構築されたデスクトップアプリケーションです。CodeMirror 6によるリッチな編集体験と、リアルタイムプレビュー、アウトライン表示などの基本的な機能に加え、以下の3つの独自機能を提供します。

## 3. 主要機能 (Key Features)

### 3.1. ブロック管理 (Block Management)
#### 機能概要 (Feature Overview)
Markdownドキュメントを意味のあるブロック（例: 見出し、リスト、コードブロックなど）に分割し、これらのブロックをドラッグ＆ドロップで直感的に並べ替えることができる機能です。これにより、長文のドキュメント構造を容易に再構築し、コンテンツの整理を効率化できます。

#### 操作方法 (How to Use)
1. エディター上でMarkdownコンテンツを作成します。
2. 各Markdownブロックの左側に表示されるハンドルをドラッグすることで、ブロックを上下に移動させることができます。
3. ブロックを目的の位置にドロップすると、ドキュメントの構造がリアルタイムで更新されます。

#### 技術的詳細 (Technical Details)
CodeMirror 6のState ManagementとDecorationsを活用し、Markdown AST (Abstract Syntax Tree) に基づいてブロックを識別・管理しています。ドラッグ＆ドロップ操作は、Reactのコンポーネントと連携してUIを更新します。

### 3.2. 変数システム (Variable System)
#### 機能概要 (Feature Overview)
`{{variable_name}}` の形式で変数を定義し、ドキュメント内で使用できる機能です。これにより、動的なコンテンツやテンプレートの作成が可能となり、特にLLM (Large Language Model) のプロンプト作成などにおいて、再利用性と柔軟性を高めます。

#### 操作方法 (How to Use)
1. ドキュメント内の任意の場所で `{{変数名}}` の形式で変数を記述します。
2. エディターのサイドバーまたは設定パネルで、定義された変数の値を入力・管理できます。
3. プレビュー画面では、変数がその値に置換された状態で表示されます。

#### 技術的詳細 (Technical Details)
CodeMirror 6の拡張機能として、変数パターンを検出し、その値を管理するシステムを実装しています。変数の解決と置換は、プレビューレンダリング時に行われます。

### 3.3. 構文チェック (Syntax Checking)
#### 機能概要 (Feature Overview)
GitHub Flavored Markdown (GFM) の仕様に準拠しているかをリアルタイムでチェックし、構文エラーや警告をエディター上に表示する機能です。これにより、記述ミスを早期に発見し、一貫性のあるMarkdownドキュメントの作成を支援します。

#### 操作方法 (How to Use)
1. Markdownを記述すると、GFMの仕様に違反する箇所がエディター上でハイライト表示されます。
2. エラー箇所にカーソルを合わせると、具体的なエラー内容のツールチップが表示されます。
3. エディター下部のパネルには、ドキュメント全体の構文エラー一覧が表示されます。

#### 技術的詳細 (Technical Details)
Markdownパーサーと連携し、AST解析中にGFMのルールに照らして構文を検証します。エラー情報はCodeMirror 6のDiagnostics機能を通じてエディターにフィードバックされます。

## 4. その他の機能 (Other Features)
-   **リアルタイムプレビュー**: 編集内容が即座にレンダリングされるプレビューパネル。
-   **シンタックスハイライト**: CodeMirror 6による多言語対応のシンタックスハイライト。
-   **ファイル操作**: ドキュメントの新規作成、保存、開くなどの基本的なファイル操作。
-   **テーマ設定**: UIのテーマ（ライト/ダークなど）を切り替える機能。
-   **パネルリサイズ**: エディターとプレビューパネルの幅を調整する機能。
